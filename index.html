Smart ROAS — V1 Campaign Optimisation Algorithm

Target-ROAS auto-managed campaigns · 1 offer_code = 1 campaign · 3-day evaluation loop

Step 1 — Campaign decision matrix (BU% × ROAS%)

Every 3 days, each campaign is assessed on two signals computed over the trailing 3-day window: ROAS% = actual ROAS ÷ target ROAS, and BU% = spend ÷ daily budget. The cell determines one intent: TIGHTEN, EXPAND, or HOLD.

Step 0 — Daily circuit breaker

Runs every day, independent of the 3-day loop. A safety guardrail, not an optimiser — fires only in the clear-disaster case.

Trigger: BU% > 80% AND ROAS% < 80%
Campaign is spending its full budget at bad efficiency — real money lost, can't wait for the 3-day cycle. BU > 80% also self-gates against noise (can't spend a full budget on a handful of clicks).
Action: one-time aggressive cut on all keyword bids
A single large cut across the campaign. Not repeated daily — fires once when the condition is met, then the 3-day loop takes over.

Deep dive — TIGHTEN

Campaign ROAS% < 80%. The campaign pays too much per conversion. Cuts are surgical, not flat — keywords are split into three tiers by data sufficiency and acted on at the smallest unit with a trustworthy signal.

Tier 1 — High-data keywords
clicks ≥ MIN_CLICKS — judge individually
  • Keyword ROAS < target → cut bid, proportional to the miss
  • Keyword ROAS ≥ target → hold (don't punish a good keyword for the campaign)
  • Mostly Established + Mid-volume keywords
Tier 2 — Long-tail block
clicks < MIN_CLICKS — judge collectively
  • Pool all low-data keywords into one block
  • Compute the block's collective ROAS
  • Block ROAS < target → one uniform modest cut across the block
  • Never judge a single long-tail keyword
Tier 3 — Zero-data keywords
no signal at all
  • Untouched
  • No basis to act — leave the bid as is

Proportional cut (Tier 1) — ratio-based, with per-cycle cap

ROAS and bid are inversely related (ROAS ∝ 1/bid). To move a keyword's ROAS to target, the bid scales by the ratio of current ROAS to target ROAS.

raw_cut = 1 − (keyword_ROAS / target_ROAS)   // implied cut to hit target cut_factor = min(raw_cut, MAX_CUT_PER_CYCLE)   // cap one cycle's cut new_bid = current_bid × (1 − cut_factor)

Clamp (all tiers)

new_bid = max(new_bid, bid_floor)   // floor-only — V1 only ever cuts
The cut is derived from the actual bid↔ROAS relationship, not a tuned coefficient — a keyword at 37.5% of target ROAS implies its bid should drop to 37.5% of current. The cap (MAX_CUT_PER_CYCLE) bounds how far a bid can fall in one 3-day cycle, so a very bad keyword converges over a few cycles instead of one violent swing. The circuit breaker is exempt from the cap. V1 has no bid-recovery step — bids only ratchet down, so the clamp only needs a floor (a cut can never raise a bid above its current value).

Deep dive — EXPAND

Campaign ROAS% ≥ 80% AND BU% < 80%. The campaign is healthy but not spending its full budget. Low budget use is a coverage problem, not a bid problem — EXPAND adds keywords and does not touch bids.

Source candidates
Pull organic keywords for this offer_code from product_performance_aggregated — organic only (keywords not established on ads).
Filter
Keep a candidate only if: organic views ≥ MIN_ORG_VIEWS (kills noise — proven demand), organic CVR ≥ MIN_CVR (quality gate), and the keyword is not already in the campaign (genuine add).
Rank
Sort survivors by organic orders, descending. Orders — not raw CVR — so tiny high-variance keywords don't float to the top.
Add top N
Add the top N keywords to the campaign at a starting bid equal to the campaign's current average keyword bid.
CVR is a filter, not the sort key. Ranking by CVR alone surfaces low-volume keywords with extreme ratios. Ranking by organic orders favours keywords that are both high-demand and converting.

HOLD

Campaign ROAS% ≥ 80% AND BU% > 80% — on or above target, spending the full budget. No action. The campaign config is held for the next 3-day cycle.

V1 scope notes

· First optimisation runs ~Day 4 (needs a 3-day window). Days 2–3 are pure learning; only the circuit breaker is active.

· No keyword removal in V1 — bad keywords are handled by aggressive bid cuts, not by negative targets.

· No exploration/performance budget split in V1 — the entire daily budget is performance budget.

· No bid recovery — bids only ratchet down. Campaigns will need periodic resets, or recovery added in V2.

· All joins on offer_code × country. Target ROAS and daily budget come from the campaign_settings table.

/* ------------------------------------------------------------
-- KEYWORD PERFORMANCE — version (B): REAL QUERY  [COMMENTED OUT]
-- The product_ads_events keyword query, adjusted for the algorithm:
--   * offer_code + country added to the grouping (algorithm is
--     per-campaign; the original query pooled keywords across campaigns)
--   * window = last eval_window_days (was current_date()-1)
--   * output columns renamed to match version (A):
--     offer_code, country, keyword, views, clicks, spend_usd, gmv_usd
-- To go live: comment out version (A), uncomment this block.
keyword_performance as (
  with
  country_curr as (
    select *,
      case
        when code = 'AED' then 'ae'
        when code = 'SAR' then 'sa'
        when code = 'EGP' then 'eg'
      end as country
    from `noondwh.ref.currency`
  ),
  purchases as (
    select
      a.action_id,
      e.event_time,
      e.visitor_id,
      e.sku,
      a.tot_offer_amount as gmv
    from `noonbiadtech.attribution.indexed_attribution`
    where a.action_type = 'purchase'
      and event_date >= current_date() - eval_window_days
  )
  select
    t.offer_code,
    t.country_code              as country,
    t.target_value              as keyword,
    sum(t.views)                as views,
    sum(t.clicks)               as clicks,
    round(sum(t.spends_usd), 2) as spend_usd,
    sum(t.ad_gmv_usd)           as gmv_usd
  from (
    select
      ev.offer_code,
      c.country_code,
      ev.target_value,
      sum(case when ev.event_type = 'view'  then 1 else 0 end) as views,
      sum(case when ev.event_type = 'click' then 1 else 0 end) as clicks,
      round(sum(if(ev.event_type = 'click', ev.bid / to_usd, 0)), 2) as spends_usd,
      sum(gmv / to_usd) as ad_gmv_usd
    from `noonbiadtech.mp_ads_events.product_ads_events` ev
      join `noondwh.ads_product_admanager.adgroup`  adg using (adgroup_code)
      join `noondwh.ads_product_admanager.campaign` c   using (campaign_code)
      left join purchases using (event_time, visitor_id, sku)
      left join country_curr cr on cr.country = c.country_code
    where c.campaign_type = 'product'
      and ev.event_date >= current_date() - eval_window_days
      and ev.targeting_type = 'keyword'
      and ev.ad_type in ('plp','merch_plp','pdp','slim_pdp','plpdeal','merchdeal','cart_product')
    group by 1, 2, 3, ev.event_date
  ) as t
  group by 1, 2, 3
),
------------------------------------------------------------ */

/* ------------------------------------------------------------
-- ORGANIC KEYWORDS — version (B): REAL QUERY  [COMMENTED OUT]
-- From product_performance_aggregated, all organic keyword activity
-- for each offer_code x country.
--   organic_views  = views  - ads_views   (clipped at 0)
--   organic_clicks = clicks - ads_clicks  (clipped at 0)
--   organic_orders ~ purchases (combined ads+organic; for keywords
--     not in the campaign this is effectively all organic)
-- No "organic only" filter here: keywords already in the campaign
-- are removed later by the anti-join in expand_candidates
-- (where cs.keyword is null) - the exact source of truth, not a proxy.
-- To go live: comment out version (A), uncomment this block.
organic_keywords as (
  select
    offer_code,
    country_code as country,
    search_query as keyword,
    greatest(sum(views)  - sum(ads_views),  0) as organic_views,
    greatest(sum(clicks) - sum(ads_clicks), 0) as organic_clicks,
    sum(purchases)                             as organic_orders
  from `noonbiadtech.organic_metrics.product_performance_aggregated`
  where event_date >= current_date() - eval_window_days
  group by offer_code, country_code, search_query
),
------------------------------------------------------------ */

-- ============================================================
-- CAMPAIGN SIGNALS + MATRIX INTENT
-- ============================================================
campaign_perf as (
  select offer_code, country,
         sum(spend_usd) as campaign_spend,
         sum(gmv_usd)   as campaign_gmv
  from keyword_performance
  group by offer_code, country
),

campaign_intent as (
  select
    s.offer_code,
    s.country,
    s.target_roas,
    s.daily_budget,
    safe_divide(safe_divide(cp.campaign_gmv, cp.campaign_spend), s.target_roas) as roas_pct,
    safe_divide(cp.campaign_spend, s.daily_budget * eval_window_days)           as bu_pct,
    case
      when safe_divide(safe_divide(cp.campaign_gmv, cp.campaign_spend), s.target_roas) < roas_low_threshold
        then 'TIGHTEN'
      when safe_divide(cp.campaign_spend, s.daily_budget * eval_window_days) < bu_threshold
        then 'EXPAND'
      else 'HOLD'
    end as intent,
    ( safe_divide(cp.campaign_spend, s.daily_budget * eval_window_days) > bu_threshold
      and safe_divide(safe_divide(cp.campaign_gmv, cp.campaign_spend), s.target_roas) < roas_low_threshold
    ) as circuit_breaker
  from campaign_settings s
  left join campaign_perf cp using (offer_code, country)
),

-- ============================================================
-- BRANCH A — EXISTING KEYWORDS (circuit breaker / TIGHTEN / HOLD)
-- ============================================================
keyword_metrics as (
  select
    cs.offer_code, cs.country, cs.keyword,
    cs.current_bid,
    coalesce(kp.clicks, 0)    as clicks,
    coalesce(kp.spend_usd, 0) as spend_usd,
    coalesce(kp.gmv_usd, 0)   as gmv_usd,
    safe_divide(kp.gmv_usd, kp.spend_usd) as keyword_roas,
    case
      when coalesce(kp.clicks, 0) >= min_clicks then 'TIER1_INDIVIDUAL'
      when coalesce(kp.clicks, 0) > 0           then 'TIER2_LONGTAIL'
      else                                           'TIER3_NODATA'
    end as tier
  from campaign_structure cs
  left join keyword_performance kp using (offer_code, country, keyword)
),

longtail_block as (
  select offer_code, country,
         safe_divide(sum(gmv_usd), sum(spend_usd)) as block_roas
  from keyword_metrics
  where tier = 'TIER2_LONGTAIL'
  group by offer_code, country
),

existing_keyword_output as (
  select
    km.offer_code,
    km.country,
    km.keyword,

    -- cut factor by branch
    --   ratio-based: ROAS proportional to 1/bid, so bid scales by keyword_ROAS/target_ROAS
    case
      when ci.circuit_breaker                                              then cb_cut
      when ci.intent != 'TIGHTEN'                                          then 0
      when km.tier = 'TIER1_INDIVIDUAL' and km.keyword_roas < ci.target_roas
        then least(1 - safe_divide(km.keyword_roas, ci.target_roas),
                   max_cut_per_cycle)
      when km.tier = 'TIER2_LONGTAIL' and lb.block_roas < ci.target_roas    then block_cut
      else 0
    end as cut_factor,

    km.current_bid,
    km.tier,
    km.keyword_roas,
    ci.target_roas,
    ci.intent,
    ci.circuit_breaker,
    lb.block_roas
  from keyword_metrics km
  join campaign_intent ci     using (offer_code, country)
  left join longtail_block lb using (offer_code, country)
),

branch_a as (
  select
    offer_code,
    country,
    keyword,
    -- final bid: apply cut, clamp at bid_floor (V1 only cuts, so no upper clamp needed)
    round(
      greatest(current_bid * (1 - cut_factor), bid_floor),
      4
    ) as bid,
    -- action tag
    case
      when circuit_breaker                                       then 'CIRCUIT_BREAKER_CUT'
      when intent = 'HOLD'                                       then 'HOLD'
      when intent = 'EXPAND'                                     then 'HOLD_BID'   -- EXPAND adds kws, leaves bids
      when tier = 'TIER3_NODATA'                                 then 'HOLD_NO_DATA'
      when tier = 'TIER1_INDIVIDUAL' and keyword_roas >= target_roas
                                                                 then 'HOLD_ON_TARGET'
      when tier = 'TIER1_INDIVIDUAL'                             then 'CUT_PROPORTIONAL'
      when tier = 'TIER2_LONGTAIL' and block_roas < target_roas  then 'CUT_LONGTAIL_BLOCK'
      when tier = 'TIER2_LONGTAIL'                               then 'HOLD_LONGTAIL_OK'
      else 'HOLD'
    end as tag
  from existing_keyword_output
),

-- ============================================================
-- BRANCH B — NEW KEYWORDS (EXPAND additions)
-- ============================================================
campaign_avg_bid as (
  select offer_code, country, round(avg(current_bid), 4) as avg_bid
  from campaign_structure
  group by offer_code, country
),

expand_candidates as (
  select
    ok.offer_code,
    ok.country,
    ok.keyword,
    cab.avg_bid as starting_bid,
    row_number() over (
      partition by ok.offer_code, ok.country
      order by ok.organic_orders desc, ok.organic_views desc
    ) as rn
  from organic_keywords ok
  join campaign_intent ci   using (offer_code, country)
  join campaign_avg_bid cab using (offer_code, country)
  -- not already in the campaign
  left join campaign_structure cs
    on  cs.offer_code = ok.offer_code
    and cs.country    = ok.country
    and cs.keyword    = ok.keyword
  where ci.intent = 'EXPAND'                                  -- only EXPAND campaigns
    and cs.keyword is null                                    -- genuine new keyword
    and ok.organic_views >= min_org_views                     -- noise floor
    and safe_divide(ok.organic_orders, ok.organic_clicks) >= min_cvr   -- quality floor
),

branch_b as (
  select
    offer_code,
    country,
    keyword,
    starting_bid as bid,
    'ADD_EXPAND' as tag
  from expand_candidates
  where rn <= expand_top_n
)

-- ============================================================
-- UNIFIED OUTPUT
-- ============================================================
select offer_code, country, keyword, bid, tag from branch_a
union all
select offer_code, country, keyword, bid, tag from branch_b
order by offer_code, country,
         case when tag = 'ADD_EXPAND' then 2 else 1 end,   -- existing kws first, added kws after
         keyword;
