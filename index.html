Smart ROAS — V1 Campaign Optimisation Algorithm

Target-ROAS auto-managed campaigns · 1 offer_code = 1 campaign · 3-day evaluation loop

Campaign decision matrix (BU% × ROAS%)

Every 3 days, each campaign is assessed on two signals computed over the trailing 3-day window: ROAS% = actual ROAS ÷ target ROAS, and BU% = spend ÷ daily budget. The cell determines one intent: TIGHTEN, EXPAND, or HOLD.

Keyword adding is separate from intent — it runs whenever BU% < 80% (the whole top row), for TIGHTEN and EXPAND alike.

Daily circuit breaker — deferred to V2

A daily safety check — BU% > 80% AND ROAS% < 80% → one-time aggressive cut — was designed to catch a campaign burning its full budget at bad efficiency without waiting for the 3-day cycle.

Not in V1. It is a genuinely separate job: a daily query on a 1-day window, distinct from this 3-day optimisation loop. Folding it into the unified 3-day query would make it a 3-day condition, not a daily safety net — which defeats its purpose.

V1 ships with the 3-day loop only. The circuit breaker is a V2 item: a standalone daily query running just the emergency-cut branch. The unified SQL below does not include it.

Deep dive — TIGHTEN

Campaign ROAS% < 80%. The campaign pays too much per conversion. Cuts are surgical, not flat — keywords are split into three tiers by data sufficiency and acted on at the smallest unit with a trustworthy signal.

Tier 1 — High-data keywords
clicks ≥ MIN_CLICKS — judge individually
  • Keyword ROAS < target → cut bid, proportional to the miss
  • Keyword ROAS ≥ target → hold (don't punish a good keyword for the campaign)
  • Mostly Established + Mid-volume keywords
Tier 2 — Long-tail block
clicks < MIN_CLICKS — judge collectively
  • Pool all low-data keywords into one block
  • Compute the block's collective ROAS
  • Block ROAS < target → one uniform modest cut across the block
  • Never judge a single long-tail keyword
Tier 3 — Zero-data keywords
no signal at all
  • Untouched
  • No basis to act — leave the bid as is

Bid cut — Tier 1, ratio-based

cut_factor = min( 1 − keyword_ROAS / target_ROAS , MAX_CUT_PER_CYCLE ) new_bid = max( current_bid × (1 − cut_factor) , bid_floor )

ROAS ∝ 1/bid, so the bid scales by keyword_ROAS / target_ROAS. A keyword at 37.5% of target → bid drops to ~37.5% of current. The cap limits how far a bid can fall in one cycle; big misses converge over several cycles.

Deep dive — keyword adding

Keyword adding is driven by BU% < 80% (budget headroom), not by intent — it runs for any campaign with budget headroom, whether TIGHTEN or EXPAND. A TIGHTEN campaign with BU% > 80% (already spending its full budget) does not add keywords. Low budget use is usually a coverage problem (too few keywords), so the fix is to add proven keywords.

Source candidates
Pull organic keywords for this offer_code from product_performance_aggregated (30-day window).
Filter
Keep a candidate only if: organic views ≥ MIN_ORG_VIEWS (kills noise — proven demand) and organic CVR ≥ MIN_CVR (quality gate).
Rank → fixed top-N pool
Sort by organic orders descending; the candidate pool is the top ORGANIC_POOL_N (10). Orders, not raw CVR — so tiny high-variance keywords don't float up.
Add up to N per cycle
Add up to ADD_PER_CYCLE (5) keywords from the pool that aren't already in the campaign, at a flat ADD_START_BID. Once the campaign holds the whole top-10 pool, nothing more is added — adding stops on its own.
The fixed top-10 pool is the natural stop condition: adding draws only from it, so once exhausted, the campaign is done — no separate cap or counter needed. New keywords start at a low flat bid (ADD_START_BID); if they convert they survive, if not the next TIGHTEN cycle cuts them.

Deep dive — EXPAND

Campaign ROAS% ≥ 80% AND BU% < 80% — healthy, with budget headroom. EXPAND always adds keywords (see keyword adding above, which it shares with TIGHTEN). In addition, when the campaign is over-delivering — ROAS% > 120% — it raises bids on Tier 1 keywords to put the spare budget to work on proven winners.

Bid raise — Tier 1, only when campaign ROAS% > 120%

raise_factor = min( keyword_ROAS / target_ROAS − 1 , MAX_RAISE_PER_CYCLE ) new_bid = min( current_bid × (1 + raise_factor) , bid_ceiling )

Mirror of the TIGHTEN cut. Only fires for over-delivering campaigns (ROAS% > 120%), so there is ROAS headroom before a higher bid pulls ROAS back to target. The raise cap is set below the cut cap — a bad raise overspends, so steps are smaller. Long-tail and no-data tiers are never raised. The ROAS% 80–120% EXPAND cell adds keywords only — no bid raise, since there is no headroom above target.

HOLD

Campaign ROAS% ≥ 80% AND BU% > 80% — on or above target, spending the full budget. No action. The campaign config is held for the next 3-day cycle.

V1 scope notes

· First optimisation runs ~Day 4 (needs a 3-day window). Days 1–3 are pure learning — no actions taken.

· The daily circuit breaker is deferred to V2 — see the flagged section above. V1 is the 3-day loop only.

· No keyword removal in V1 — bad keywords are handled by aggressive bid cuts, not by negative targets.

· No exploration/performance budget split in V1 — the entire daily budget is performance budget.

· Bids cut for underperformers, raised only for over-delivering campaigns (ROAS > 120%). Raise cap is tighter than cut cap.

· All joins on offer_code × country. Target ROAS and daily budget come from the campaign_settings table.

The full algorithm as one BigQuery query. Output: one row per offer_code × country × keyword, with the bid to apply and the action tag. /* ------------------------------------------------------------
-- KEYWORD PERFORMANCE — version (B): REAL QUERY  [COMMENTED OUT]
-- The product_ads_events keyword query, adjusted for the algorithm:
--   * offer_code + country added to the grouping (algorithm is
--     per-campaign; the original query pooled keywords across campaigns)
--   * window = last eval_window_days (was current_date()-1)
--   * output columns renamed to match version (A):
--     offer_code, country, keyword, views, clicks, spend_usd, gmv_usd
-- To go live: comment out version (A), uncomment this block.
keyword_performance as (
  with
  country_curr as (
    select *,
      case
        when code = 'AED' then 'ae'
        when code = 'SAR' then 'sa'
        when code = 'EGP' then 'eg'
      end as country
    from `noondwh.ref.currency`
  ),
  purchases as (
    select
      a.action_id,
      e.event_time,
      e.visitor_id,
      e.sku,
      a.tot_offer_amount as gmv
    from `noonbiadtech.attribution.indexed_attribution`
    where a.action_type = 'purchase'
      and event_date >= current_date() - eval_window_days
  )
  select
    t.offer_code,
    t.country_code              as country,
    t.target_value              as keyword,
    sum(t.views)                as views,
    sum(t.clicks)               as clicks,
    round(sum(t.spends_usd), 2) as spend_usd,
    sum(t.ad_gmv_usd)           as gmv_usd
  from (
    select
      ev.offer_code,
      c.country_code,
      ev.target_value,
      sum(case when ev.event_type = 'view'  then 1 else 0 end) as views,
      sum(case when ev.event_type = 'click' then 1 else 0 end) as clicks,
      round(sum(if(ev.event_type = 'click', ev.bid / to_usd, 0)), 2) as spends_usd,
      sum(gmv / to_usd) as ad_gmv_usd
    from `noonbiadtech.mp_ads_events.product_ads_events` ev
      join `noondwh.ads_product_admanager.adgroup`  adg using (adgroup_code)
      join `noondwh.ads_product_admanager.campaign` c   using (campaign_code)
      left join purchases using (event_time, visitor_id, sku)
      left join country_curr cr on cr.country = c.country_code
    where c.campaign_type = 'product'
      and ev.event_date >= current_date() - eval_window_days
      and ev.targeting_type = 'keyword'
      and ev.ad_type in ('plp','merch_plp','pdp','slim_pdp','plpdeal','merchdeal','cart_product')
    group by 1, 2, 3, ev.event_date
  ) as t
  group by 1, 2, 3
),
------------------------------------------------------------ */

-- ORGANIC KEYWORDS — version (B): REAL QUERY  [COMMENTED OUT]
-- From product_performance_aggregated, all organic keyword activity
-- for each offer_code x country.
--   organic_views  = views  - ads_views   (clipped at 0)
--   organic_clicks = clicks - ads_clicks  (clipped at 0)

    offer_code,

-- ============================================================
-- CAMPAIGN SIGNALS + MATRIX INTENT
-- ============================================================
campaign_perf as (
  select offer_code, country,
         sum(spend_usd) as campaign_spend,
         sum(gmv_usd)   as campaign_gmv
  from keyword_performance
  group by offer_code, country
),

campaign_intent as (
  select
    s.offer_code,
    s.country,
    s.target_roas,
    s.daily_budget,
    safe_divide(safe_divide(cp.campaign_gmv, cp.campaign_spend), s.target_roas) as roas_pct,
    safe_divide(cp.campaign_spend, s.daily_budget * eval_window_days)           as bu_pct,
    case
      when safe_divide(safe_divide(cp.campaign_gmv, cp.campaign_spend), s.target_roas) < roas_low_threshold
        then 'TIGHTEN'
      when safe_divide(cp.campaign_spend, s.daily_budget * eval_window_days) < bu_threshold
        then 'EXPAND'
      else 'HOLD'
    end as intent
  from campaign_settings s
  left join campaign_perf cp using (offer_code, country)
),

-- ============================================================
-- BRANCH A — EXISTING KEYWORDS (TIGHTEN cuts / EXPAND raises / HOLD)
-- ============================================================
keyword_metrics as (
  select
    cs.offer_code, cs.country, cs.keyword,
    cs.current_bid,
    coalesce(kp.clicks, 0)    as clicks,
    coalesce(kp.spend_usd, 0) as spend_usd,
    coalesce(kp.gmv_usd, 0)   as gmv_usd,
    safe_divide(kp.gmv_usd, kp.spend_usd) as keyword_roas,
    case
      when coalesce(kp.clicks, 0) >= min_clicks then 'TIER1_INDIVIDUAL'
      when coalesce(kp.clicks, 0) > 0           then 'TIER2_LONGTAIL'
      else                                           'TIER3_NODATA'
    end as tier
  from campaign_structure cs
  left join keyword_performance kp using (offer_code, country, keyword)
),

longtail_block as (
  select offer_code, country,
         safe_divide(sum(gmv_usd), sum(spend_usd)) as block_roas
  from keyword_metrics
  where tier = 'TIER2_LONGTAIL'
  group by offer_code, country
),

existing_keyword_output as (
  select
    km.offer_code,
    km.country,
    km.keyword,

    -- bid multiplier: <1 cut, >1 raise, =1 hold
    --   ratio-based: ROAS proportional to 1/bid, so bid scales by keyword_ROAS/target_ROAS
    case
      -- TIGHTEN: cut underperforming keywords
      when ci.intent = 'TIGHTEN'
           and km.tier = 'TIER1_INDIVIDUAL'
           and km.keyword_roas < ci.target_roas
        then 1 - least(1 - safe_divide(km.keyword_roas, ci.target_roas),
                       max_cut_per_cycle)
      when ci.intent = 'TIGHTEN'
           and km.tier = 'TIER2_LONGTAIL'
           and lb.block_roas < ci.target_roas
        then 1 - block_cut

      -- EXPAND + campaign over-delivering (ROAS% > 120%): raise Tier 1 keywords
      when ci.intent = 'EXPAND'
           and ci.roas_pct > roas_high_threshold
           and km.tier = 'TIER1_INDIVIDUAL'
           and km.keyword_roas > ci.target_roas
        then 1 + least(safe_divide(km.keyword_roas, ci.target_roas) - 1,
                       max_raise_per_cycle)

      -- everything else: hold
      else 1
    end as bid_multiplier,

    km.current_bid,
    km.tier,
    km.keyword_roas,
    ci.target_roas,
    ci.roas_pct,
    ci.intent,
    lb.block_roas
  from keyword_metrics km
  join campaign_intent ci     using (offer_code, country)
  left join longtail_block lb using (offer_code, country)
),

branch_a as (
  select
    offer_code,
    country,
    keyword,
    -- apply multiplier, then clamp to [bid_floor, bid_ceiling]
    round(
      least(
        greatest(current_bid * bid_multiplier, bid_floor),
        bid_ceiling
      ), 4
    ) as bid,
    -- action tag
    case
      when intent = 'HOLD'                                       then 'HOLD'
      when tier = 'TIER3_NODATA'                                 then 'HOLD_NO_DATA'
      when intent = 'TIGHTEN' and tier = 'TIER1_INDIVIDUAL' and keyword_roas >= target_roas
                                                                 then 'HOLD_ON_TARGET'
      when intent = 'TIGHTEN' and tier = 'TIER1_INDIVIDUAL'      then 'CUT_PROPORTIONAL'
      when intent = 'TIGHTEN' and tier = 'TIER2_LONGTAIL' and block_roas < target_roas
                                                                 then 'CUT_LONGTAIL_BLOCK'
      when intent = 'TIGHTEN' and tier = 'TIER2_LONGTAIL'        then 'HOLD_LONGTAIL_OK'
      when intent = 'EXPAND'  and roas_pct > roas_high_threshold
           and tier = 'TIER1_INDIVIDUAL' and keyword_roas > target_roas
                                                                 then 'RAISE_PROPORTIONAL'
      when intent = 'EXPAND'                                     then 'HOLD_BID'   -- EXPAND adds kws
      else 'HOLD'
    end as tag
  from existing_keyword_output
),

-- ============================================================
-- BRANCH B — NEW KEYWORDS (added when BU% < 80%, any intent)
-- ============================================================
-- Candidate pool = top N organic keywords for the offer_code,
-- ranked over product_performance_aggregated's full 30-day window.
-- Each cycle, add up to add_per_cycle of the pool that are not
-- already in the campaign. Once all top-N are in, the pool is
-- exhausted and nothing more is added - self-saturating, no cap needed.

-- rank the full organic candidate set, then keep only the top-N pool
organic_pool as (
  select
    offer_code,
    country,
    keyword,
    row_number() over (
      partition by offer_code, country
      order by organic_orders desc, organic_views desc
    ) as pool_rank
  from organic_keywords
  where organic_views >= min_org_views                          -- noise floor
    and safe_divide(organic_orders, organic_clicks) >= min_cvr   -- quality floor
  qualify pool_rank <= organic_pool_n
),

-- from the pool, keep only keywords not already in the campaign,
-- for campaigns with budget headroom (BU% < 80%)
add_candidates as (
  select
    op.offer_code,
    op.country,
    op.keyword,
    row_number() over (
      partition by op.offer_code, op.country
      order by op.pool_rank
    ) as add_rank
  from organic_pool op
  join campaign_intent ci using (offer_code, country)
  left join campaign_structure cs
    on  cs.offer_code = op.offer_code
    and cs.country    = op.country
    and cs.keyword    = op.keyword
  where ci.bu_pct < bu_threshold        -- budget headroom (TIGHTEN or EXPAND)
    and cs.keyword is null              -- not already in the campaign
),

branch_b as (
  select
    offer_code,
    country,
    keyword,
    add_start_bid as bid,
    'ADD_KEYWORD' as tag
  from add_candidates
  where add_rank <= add_per_cycle       -- add up to N this cycle
)

-- ============================================================
-- UNIFIED OUTPUT
-- ============================================================
select offer_code, country, keyword, bid, tag from branch_a
union all
select offer_code, country, keyword, bid, tag from branch_b
order by offer_code, country,
         case when tag = 'ADD_KEYWORD' then 2 else 1 end,   -- existing kws first, added kws after
         keyword;
